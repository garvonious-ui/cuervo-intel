Typography

Two families. Barlow Condensed is the global default — registered at the Streamlit framework level via .streamlit/config.toml so every native widget inherits it. Inter is used only for per-component overrides where condensed type would hurt legibility (body copy, captions, hero subtitles, CPN tables).
Barlow Condensed
Role: Headlines, section labels, KPI values, pillar names, hero titles, all uppercase editorial type.
Weights: 400, 500, 600, 700, 800
Registered: .streamlit/config.toml → [[theme.fontFaces]] (5 weights)
Fallback stack: Barlow Condensed, Helvetica, Arial, sans-serif

Inter
Role: Body copy, captions, hero subtitles, tables, card descriptions, metadata, status pills.
Weights used: 400, 500, 600, 700
Registered: Via CSS font-family rules on specific components (never as global default).

auto_awesome

Voice

Poplife's editorial tone for the dashboard itself — not client-facing content. Direct, warm, specific, zero corporate filler.
Do
  • Name things plainly — "Brand Performance," not "Strategic Efficacy Insights."
  • Use editorial code names once the user is on the page ("THE MIRROR", "THE WINDOW") as payoff, not navigation.
  • Lead metrics with a specific number + what's surprising about it.
  • Write callouts like a smart colleague explaining the thing, not a slide deck.
Don't
  • Don't use consulting-speak ("leverage", "synergies", "drive outcomes").
  • Don't hide bad numbers — the scorecard's job is to surface honest gaps.
  • Don't use emoji in product UI unless a client deck specifically calls for it.
  • Don't write copy that would look the same for any client — specificity builds trust.

Implementation gotchas

Hard-won lessons from shipping Treatment C across 5 Streamlit pages. Read these before building any new component.

1. Fonts go in .streamlit/config.toml, not CSS

Register custom fonts via [[theme.fontFaces]] blocks and set font = "..." under [theme]. Streamlit applies theme settings AFTER CSS injection, so CSS font-family rules lose even with !important. Use CSS only for per-component overrides (e.g., Inter on hero subtitles, content-card tables). See docs/decisions.md (2026-04-09).

2. Use <div>, not <h1>–<h6>, inside flex/grid HTML

Streamlit has a heading interceptor that wraps every <h*> in markdown output with <div data-testid="stHeadingWithActionElements"> for anchor linking, hoisting it out of its parent. This breaks display: flex; justify-content: space-between layouts. Use <div class="sku-name"> styled via CSS instead of <h4>. See docs/decisions.md (2026-04-14).

3. render_content_card_open/close only works for pure HTML

Streamlit widgets (st.selectbox, st.button, st.plotly_chart, st.dataframe) render as siblings to markdown output, not children. The open <div class="content-card"> gets auto-closed at the end of the st.markdown block. For widget content, use a .section-label + caption instead of wrapping.

4. No 4+ leading spaces in st.markdown HTML f-strings

Streamlit's markdown parser treats 4+ leading spaces as a code block, silently dropping nested HTML. Build complex nested layouts as single-line f-strings.

5. Material Symbols icons need a CSS override when global font is set

Setting font = "Barlow Condensed" cascades down to Streamlit's icon spans ([data-testid="stIconMaterial"]), breaking ligatures. Pin them back to Material Symbols Rounded with font-feature-settings: 'liga' in your CSS. Already handled in POPLIFE_TREATMENT_C_CSS.

6. Streamlit Cloud auto-rebuild is best-effort

If a push doesn't appear live within ~60s, DOM-probe to confirm (don't trust the title or view source — caching is aggressive). If the app is stale, manually reboot via Manage app → Reboot. Required especially for .streamlit/config.toml changes.
